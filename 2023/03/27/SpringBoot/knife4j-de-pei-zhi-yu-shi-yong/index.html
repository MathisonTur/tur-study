Knife4j配置与使用(OpenAPI2)

SpringBoot
发布日期:   2023-03-27
更新日期:   2023-03-28
文章字数:   551
阅读时长:   2 分
阅读次数:  

1. 引入

maven 坐标如下

<dependency>
    <groupId>com.github.xiaoymin</groupId>
    <artifactId>knife4j-openapi2-spring-boot-starter</artifactId>
    <version>4.1.0</version>
    <!-- 具体版本自行选择 -->
</dependency>

Spring Boot 版本建议在 2.4.0 ~ 3.0.0 之间

2. 配置 yml 属性

knife4j:
  enable: true
  openapi:
    title: 这是文档标题 # 文档标题
    description: 这是简介 # 简介
    email: 123456789@qq.com # 创建人邮件
    concat: tur # 创建者
    url: https://docs.xiaominfo.com # 项目文档地址
    version: v4.0 # 版本
    license: Apache 2.0 # 许可证
    license-url: https://stackoverflow.com/ # 许可证地址
    terms-of-service-url: https://stackoverflow.com/
    group: # 分组
      test1: # 组名，用以区分其他组
        group-name: 分组名称 # 组别名称
        api-rule: package # api扫描方式
        api-rule-resources: # 指定扫描的位置
          - com.knife4j.demo.new3 # 存放接口、对象的包

访问 Knife4j 的文档地址：http://ip:sort/doc.html

3. Java注解

1. @Api：定义接口分组名称

参数说明：

  • tags:说明该类的作用，可以在 UI 界面上看到的注解
  • value：一般会自动获取接口分组的名称，可以不用设置
2. @ApiImplicitParam：单个参数注释

用在 @ApiImplicitParams 中，指定一个请求参数的各个方面

  • name：参数名
  • value：参数说明
  • required：参数时是否必须传
  • paramType：参数位置，有以下可选值
    • header：请求头 @RequestHeader
    • query：?param=value格式 @RequestParam
    • path：路径，Restful风格接口 @PathVariable
    • body：请求体
    • form：以form表单的形式提交
  • dataType：参数类型，默认 String，其他值为 Integer
  • defaultValue：参数的默认值
3. @ApiImplicitParams：多个参数注释

用在请求方法上，表示一组参数的说明

4. @ApiModel：实体类定义
5. @ApiModelProperty：实体属性定义
  • value：字段说明
  • name：属性名
  • dataType：属性类型
  • required：是否必填
  • example：举例说明
  • hidden：隐藏
6. @ApiOperation：接口定义
  • value：接口说明
  • httpMethod：接口请求方式
  • response：接口返回参数类型
  • notes：接口发布说明
7. @ApiParam：参数注释
8. @ApiResponse：响应码

用在 @ApiResponses 中，一般用于表达一个错误或正确的响应信息

  • code：响应码
  • message：响应信息
  • resposne：抛出异常类
9. @ApiResponses：多个响应码

用在请求的方法上，表示一组响应
